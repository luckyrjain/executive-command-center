# RFC-004 — System Architecture

RFC-004 is the canonical architecture index for Executive Command Center. The detailed architecture is split into independently reviewable chapters.

## Architecture chapters

1. [Architectural Vision & System Context](architecture/chapter-01-vision.md)
2. [Core Platform & Service Architecture](architecture/chapter-02a-core-platform.md)
3. [Platform Runtime, Deployment & Operational Architecture](architecture/chapter-02b-runtime.md)
4. [AI Runtime Architecture](architecture/chapter-03-ai-runtime.md)
5. [Knowledge Platform & Memory Architecture](architecture/chapter-04-knowledge-platform.md)
6. [Human Attention Engine](architecture/chapter-05-attention-engine.md)
7. [Integration Platform & Connector Framework](architecture/chapter-06-integration-platform.md)
8. [Frontend Architecture & Executive Experience](architecture/chapter-07-frontend.md)
9. [Data Platform & Storage Architecture](architecture/chapter-08-data-platform.md)
10. [Security, Privacy & Local-First Architecture](architecture/chapter-09-security.md)
11. [Operations, Deployment & Platform Engineering](architecture/chapter-10-operations.md)

## Canonical terminology

- **AI Runtime** owns model routing, agent execution, prompt versioning and evaluation.
- **Knowledge Platform** owns durable memory, retrieval and knowledge relationships.
- **Human Attention Engine** owns prioritization, focus protection and recommended next actions.
- **Integration Platform** owns connectors, synchronization and external-system contracts.

## Governance

- Chapter changes are changes to RFC-004.
- Each chapter must retain RFC-004 metadata and dependency references.
- New architecture chapters require an ADR and an update to this index.
- Implementations must trace to an approved phase specification.
