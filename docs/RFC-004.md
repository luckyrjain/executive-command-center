# RFC-004 — System Architecture

RFC-004 defines the target architecture of Executive Command Center. It is split into independently reviewable chapters so each subsystem can evolve without turning the architecture into a single unmanageable document.

## Reading order

Read Chapters 1, 2A and 2B first. They establish boundaries, domain ownership, runtime behavior and deployment assumptions. The remaining chapters define specialized platform capabilities within those constraints.

## Architectural rules

- Domain ownership is explicit.
- Services communicate through versioned contracts.
- Business state belongs to domains, not AI agents.
- All model access passes through the Model Router.
- Connectors do not write directly to domain databases.
- Cross-domain database access is prohibited.
- State-changing AI actions require human approval unless a later approved phase explicitly permits otherwise.
- Local execution is the default; cloud infrastructure is an optional deployment model.

## Change control

Material changes to domain boundaries, platform ownership, data ownership, security boundaries or approved runtime patterns require an ADR and an update to the affected RFC-004 chapter.
